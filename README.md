# DangerousMemoryCacher/デンジャラスメモリキャッシャー
Aviutl用軽量化プラグイン(複数動画及び事前キャッシュ)

## 今のAviutlではPCの性能を持て余していませんか？
このプラグインでは編集している動画のキャッシュを予め先に読み込みスムーズな動画再生を促します。
## パラメータ
  * キャッシュに用いるメモリの数
  * 先読みするフレームの数
  * 一般的なログの出力の有効
  * 動画再生時に毎回連続で処理されるログの出力の有無
  * 動画出力時キャッシュ保存無効

# 警告　
  * 指定されたメモリを超えない限りフレームをキャッシュし続けるのでRAMに相当負荷がかかると思います。使用の際は自己責任でお願いします。私は責任を負えません。
### 自身のスペックにあったパラメータを設定してください。(メモリサイズをオーバーして設定するとPCはストレージを削って容量を確保し、大変危険です。)
### 保存されるキャッシュは圧縮されないので動画のファイルサイズをあてににないでください。バリバリ持ってかれます。
### 先読みフレーム数を多くすると、一回先読みするだけで設定したメモリサイズを超え、古いキャッシュから上書きされるため、結果保存されているはずの次のフレームのキャッシュが消えます。そしてまたキャッシュ保存のいたちごっこになります。
### キャッシュは動画ファイルごとに生成されます、仮に２つの動画(同じファイルでも)を同時に並べれば、メモリの消費量は2倍になります。
### 先読みは動画ファイルに応じて行われる為、カットした箇所もしっかり保存しますし、先読みの性能を発揮したいならカットされたフレーム数以上を選択するべきです。(ちょっとした無音部分などをカットするときの話)

## ダウンロードする前に…
  * 動作をさせるには[こちら](https://github.com/amate/MFVideoReader)のプラグインの導入が必要です。
  * 複数の動画再生が確実に解消されるわけではありません。
  * 動画の出力とは相性が悪いようです。プラグインの設定から「動画出力」をオンにするか、それでも遅いと感じる場合プラグインを抜いて書き出してください。
